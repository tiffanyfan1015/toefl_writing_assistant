# TOEFL 寫作小幫手實作計畫

## 目標
建立一個網頁版的 TOEFL 寫作輔助平台，支援兩種題型（Email 和 Academic Discussion）。此平台將允許使用者練習寫作、透過 Google Gemini API 獲得 AI 產生的評分與文法回饋、使用類似 Git 的差異比對介面追蹤歷次修改，並維護一個集中的文法與拼字錯誤紀錄。

## 技術堆疊
*   **前端:** 採用 Vite 建置的 React (或 Vue)
*   **後端:** Python (FastAPI)
*   **資料庫:** PostgreSQL
*   **AI 整合:** Google Gemini API
*   **使用者驗證:** 單一使用者 (不需要複雜的登入系統)

## 核心功能與範圍
1.  **題目管理:** 將多個題目分類並儲存在「Email」和「Academic Discussion」之下。
2.  **練習介面:** 分割畫面或清晰的排版，同時顯示題目與打字區。
3.  **評分與回饋:** 整合 Gemini API 來評估作答內容、給予評分，並揪出文法與拼字錯誤。
4.  **修改追蹤:** 允許使用者修改過去寫過的文章。使用視覺化的差異比對（紅/綠色標記）顯示不同版本間的變更。
5.  **錯誤紀錄庫:** 一個專屬的頁面，彙整歷史紀錄中所有的文法和拼字錯誤，以利長期學習。

## 實作步驟

### 第一階段：環境建置與資料庫設計
*   使用 Vite 初始化前端應用程式 (React 或 Vue)。
*   初始化 Python FastAPI 後端專案。
*   使用 ORM (如 SQLAlchemy 或 SQLModel) 設定 PostgreSQL 資料庫連線。
*   **資料庫 Schema 設計:**
    *   `Question` (id, 題型, 題目內容, 建立時間)
    *   `Submission` (id, question_id, 目前文章內容, 最新評分, 建立時間, 更新時間)
    *   `SubmissionRevision` (id, submission_id, 文章內容, 評分, 建立時間) - *用於差異比對追蹤*
    *   `ErrorLog` (id, submission_id, 錯誤類型, 錯誤文字, 建議修改, 解釋)

### 第二階段：後端 API 開發
*   **題目 API:** 管理 TOEFL 題目的 CRUD (新增、讀取、更新、刪除) 操作。
*   **作答 API:** 提交新文章或更新現有文章的端點。
*   **Gemini 整合:** 在 FastAPI 中建立一個服務層，負責將文章和題目發送給 Gemini API，並搭配嚴格的系統提示詞 (System Prompt)，確保回傳評分以及結構化的文法/拼字錯誤列表。
*   **差異比對邏輯:** 實作後端邏輯（或將原始字串傳遞給前端）以產生 `SubmissionRevision` 之間的行級/單字級別差異。

### 第三階段：前端開發
*   **版面配置與導覽:** 建立儀表板 (題目列表)、練習介面和錯誤紀錄庫的基本路由。
*   **練習介面:** 建立包含計時器元件的打字區 (Email 任務限制為 7 分鐘)。
*   **回饋與差異比對檢視:** 整合套件 (如 `diff-match-patch` 或 `react-diff-viewer`) 以顯示已儲存文章版本之間的紅/綠色修改標記。
*   **錯誤儀表板:** 建立資料表或列表視圖，以顯示從 `ErrorLog` 資料表中收集到的所有文法和拼字錯誤。

### 第四階段：整合與優化
*   將前端與 FastAPI 端點連接。
*   優化 Gemini API 的提示詞，以確保評分的一致性與錯誤偵測的準確度。
*   實作「儲存並修改」的流程，讓使用者可以檢閱和更新過去的文章。
